# Hephaestus GPU substrate audit — 2026-07-02

Read-only cross-repo audit of `repos/hephaestus` (core, wgpu, cuda, metal, python
crates; ~42k lines) plus a consumer survey of `repos/helios`, `repos/kwavers`,
and `CFDrs` (external, `C:/Users/RyanClanton/gcli/software/millifluidic_design/CFDrs`)
to ground the custom-kernel seam design (ADR 0004 draft).

Evidence tier: implementation audit — direct source reads with file:line
citations and measured duplicate-line counts (`comm -12`). No runtime
measurements were taken; throughput claims cite committed validation reports.
CONFIRMED = mechanism read end-to-end; PLAUSIBLE = mechanism present, impact
unmeasured. Findings already tracked in hephaestus `gap_audit.md` (host-delegated
decomposition parity, sparse perf parity, small-shape axis-reduction overhead,
CUDA-hardware verification blocked) are excluded.

## 5. Architecture (SRP/SoC/SSOT/DRY/hierarchy)

1. **[arch] Cross-backend host-orchestration duplication** — wgpu ↔ cuda `application/` trees mirror each other; measured identical-line counts: scan.rs 212/394 lines identical (54%), lu.rs 202, cholesky.rs 160, spmm.rs 82. Validation, `#[repr(C)]` meta packing, `Gpu*Decomposition` types, the ~160-line blocked-LU host loop, ZST op vocabulary (`WGSL_EXPR` vs `CUDA_EXPR` trait pairs), and public wrappers are all written twice. Estimate: **2,500–3,500 of hephaestus-cuda's ~5,000 application lines are structural duplicates**. The two blocked-LU loops have already diverged (wgpu hoists panel scratch; cuda reallocates per panel, `cuda lu.rs:250`) and the O(L²) scan defect was faithfully copied into both. Root cause: no backend-neutral dispatch/orchestration seam (→ ADR 0004). Second-occurrence consolidation trigger fired long ago; a third native backend triples the cost.
2. **[arch] Kernel seam single-implementor and under-scoped** — core `UnaryStorageKernel`/`BinaryStorageKernel` (`domain/kernel.rs:63-111`) implemented **only by wgpu** (`application/storage_kernel.rs:62,184`); construction takes raw WGSL, so it is not backend-neutral. No N-ary bind, no multi-pass encoding, no async completion, no param/push-constant contract. ADR 0001 promised the follow-up seam ADR; it was never written. This gap is the root cause of §5.1, the python fan-out (§5.3), and every consumer building a private substrate (§6).
3. **[major] hephaestus-python monolith** — 3,302 lines, 42 pyfunctions, 83 backend match arms in one file; domain logic in the binding layer (`split_packed_lu` lib.rs:1735-1753, SVD marshaling :1755, `clone_cuda_buffer` device copy via full host round-trip :1832) violates the thin-binding rule. Module split now; arm collapse follows the generic host layer. GIL discipline is good (68 `allow_threads` sites).
4. **[minor] Inconsistent hierarchy** — wgpu `linalg.rs` 2,010 lines/≥9 op families vs cuda's correct `linalg/` leaf modules; `reduction.rs` 1,765 (wgpu) and 805 (cuda) mixed-family; cuda `strided.rs` 890; wgpu `infrastructure/device.rs` 1,132. Splits are mechanical; most decomposition files shrink below 500 automatically once host orchestration hoists (§5.1). Intra-cuda: `to_i32`/`to_u32`/`map_layout_err` defined 4× in one crate (`linalg/mod.rs:67-83`, `reduction.rs:301-307`, `scan.rs:99-117`, `sparse/spmm.rs:26-32`).
5. **[minor] hephaestus-metal** — not a stub (no `todo!`; real compute via `WgpuDevice::try_metal` delegation) but 1,276 lines of pure forwarding whose substance is one constructor + a buffer newtype; the `metal = []` feature gates nothing; the name implies a native MSL backend it isn't. Decision needed: reduce to a pinned-backend constructor on WgpuDevice ([major] break) or record the alias-crate justification.
6. **[patch] Concrete-f32 decomposition surface** (`GpuLuDecomposition` hardcodes `LuDecomposition<f32>`, core `panel_lu_packed(&mut [f32])`) — grounded in leto-ops' surface but an unrecorded deviation from the Scalar-generic rule. **Edition 2021/resolver 2** with no recorded justification (current stable: 2024/3).
7. **Zero-cost inventory is otherwise strong**: zero `dyn` in workspace src (grep-verified), GAT `Buffer<T: Pod>` + PhantomData dtype typing, ZST op markers, `(TypeId, TypeId, width)` pipeline keys (wgpu), const-generic rank `StridedOperand<'a, T, const N>`, `BlockWidth` transparent NonZeroU32 newtype, honest CUDA-absent stubs (typed `AdapterUnavailable` + fail-closed runtime sanity roundtrip).

## 6. Consumer survey (seam requirements ground truth)

- **helios** (`crates/helios-gpu`): consumes hephaestus directly; its "custom kernel seam" is **upstreaming every kernel into hephaestus** (`ExpNegOp`, `ray_line_integrals` → `hephaestus-wgpu/src/application/volume.rs:40-119`). The fused affine-clamp HU→μ op is blocked today on being "absent from hephaestus-wgpu's op set" (helios `backlog.md:43`, `gap_audit.md:187`). Params are smuggled through an f32-packed storage buffer with counts constrained to exact-f32 range (`volume.rs:42-45`) because the internal harness has no uniform/param path. Needs: consumer-authored fused elementwise ops, N-storage + param-struct kernels, resident multi-kernel pipelines (gap_audit prescribes an on-device fused pipeline; elementwise alone measured transfer-bound at 0.66–0.73× CPU).
- **kwavers** (`crates/kwavers-gpu`): zero hephaestus; raw wgpu 26 + apollofft-wgpu + burn(PINN). Hand-rolled the entire substrate: device init, buffer/readback, pipeline manager, memory pool, multi-GPU context, and a bespoke `PhysicsKernelRegistry` (name → WGSL + entry + workgroup config, `backend/physics_kernels.rs:80-204`) — a private version of exactly the proposed seam. Kernel scale: `pstd.wgsl` 883 lines / **25 entry points / 31 storage buffers across 4 bind groups**, push constants (128 B, chosen over uniform re-upload), workgroup-shared-memory FFT and reductions, 5 ping-pong pairs with 3-pass barrier-dependent race-freedom design, device-resident nt-step time loop with GPU-side zeroing and one end-of-run sensor readback. GPU stack currently feature-gated "under repair against wgpu v26" (`Cargo.toml:30-33`). Also contains a delivered placeholder kernel (`operators.wgsl:84-96` spatial_derivative copies input) — an integrity item on kwavers' side.
- **CFDrs** (`cfd-core/src/compute/gpu/**`): zero hephaestus; raw wgpu with own `GpuContext`/`GpuPipelineManager`/`GpuKernel` trait + `ComputeBackend{Cpu,Simd,Gpu,Hybrid}` ladder. ~10 kernels (Laplacian w/ BC variants, advection, diffusion, Jacobi/red-black Poisson + residual, projection/divergence, turbulence). Defects the seam must not inherit: **fake-generic precision** (`T: RealField` surfaces casting `to_f32().unwrap_or(1.0)`, `cfd-math/.../gpu.rs:79-80`), per-`apply` upload/dispatch/readback round-trips inside iterative solvers, bind group + uniform re-created per execute, and an advection kernel whose `create_pipeline` builds an empty bind-group layout while `execute` returns UnsupportedOperation (`kernels/advection.rs:62-94`).

**Union requirements** (all three): kernel descriptor = param struct + N read-only + M read-write storage with per-binding access; device-resident buffer reuse across dispatches; 1D/2D/3D grids with configurable workgroup size; multi-pass single-submission chains with inter-pass barriers; f32 + CPU differential-reference hooks with an **honest** precision contract; consumer-authored kernels without forking the substrate; runtime capability ladder with surfaced reasons. Two of three: ping-pong pairs, multi-entry-point modules with per-entry pipeline caching, workgroup shared memory, push constants (with uniform fallback), complex-as-split-re/im + apollofft interop, bulk + mid-loop-scalar readback patterns, heterogeneous per-binding element types (u32 indices, vec2/vec3 packing, i8 quantized). **Explicitly out of scope** (no surveyed kernel uses them): atomics, indirect dispatch, textures/samplers, device f64, subgroup ops.

## 7. Recommended sequencing

1. **Phase 0 [patch] correctness batch** (independent of seam, small diffs): CU-C1 bind-before-launch; CU-P4 don't cache failed compiles; WG-S1 multi-device HostPinned rejection; WG-M1/M2 pool caps; WG-M6 escaped uniform guard; CU-S1/S2/WG-S3 SAFETY-comment pass + fix the cuda `lib.rs` isolation claim; WG-S7 pinv/matexp doc fix; CU-S4 NVRTC return codes.
2. **Phase 1 [arch] ADR 0004** (sign-off gate): kernel/dispatch seam in core — see `docs/adr/0004-hephaestus-kernel-seam.md` (draft). CUDA implements the storage-kernel seam as its first increment.
3. **Phase 2 [arch] consolidation**: unify ZST op vocabulary (one trait per op family with per-dialect associated consts), hoist duplicated host orchestration into a core generic layer over `D: ComputeDevice`, per op family — scan first (fixes BOTH-SCAN once), then decomposition loops, then wrappers. Python module split + arm collapse rides on this.
4. **Phase 3 [minor/patch] performance**: CUDA streams + pinned staging (CU-P1/P6/CU-M3); wgpu encoder-borrowing batching (WG-P4, subsumed by the seam's CommandStream); fused dot/norms (WG-P3); WG-P1 rank/det; file-tree splits (wgpu linalg/reduction); metal decision (§5.5).
5. **Consumers**: helios authors affine-clamp via the seam (unblocks its backlog:43); kwavers migrates its substrate onto hephaestus during its wgpu-26 repair (its registry, pipeline manager, memory pool, and multi-GPU context are all subsumed); CFDrs replaces `GpuPipelineManager`/`GpuKernel` with the seam and makes its precision contract honest.

## 8. Residual risk / unverified

- No runtime benchmarks were taken; all perf findings are mechanism-level. CU-P7 magnitude (managed-memory migration cost) and WG-P5/P9 need criterion baselines before/after per performance_engineering.
- CUDA findings are unverifiable on real hardware in this environment (known gap_audit item).
- mnemosyne CUDA allocation-registry capacity behavior (bounded → artificial `AllocationFailed` ceiling) needs a contract test (PLAUSIBLE).
- hephaestus-local items in §1–§5 are not yet synced into `repos/hephaestus/{gap_audit,backlog}.md` (submodule untouched by this read-only audit); sync is the first action of whichever phase is approved.
