    1: # Chapter 7 — Performance and Atlas Integration Overview
    2: 
    3: This Part is the **Atlas stack migration reference** for CFDrs.  It documents
    4: how the legacy third-party crates have been — or are being — replaced
    5: by the Atlas monolith: the unified, purpose-built crates that share one
    6: trait frontier ([`eunomia::RealField`]) and one set of zero-cost
    7: abstractions.
    8: 
    9: ## Performance Mandate
   10: 
   11: Atlas is held to a strict performance contract:
   12: 
   13: - **Zero heap allocations on hot paths.** All transient storage lives in
   14:   [`mnemosyne::Arena`] sub-arenas.
   15: - **Zero virtual dispatch on numeric kernels.** Trait specialization
   16:   through [`eunomia`] gives the compiler monomorphized code per
   17:   scalar type and backend.
   18: - **Zero copy for read-only views.** [`Cow`] over [`leto::NdArray`]
   19:   eliminates `clone()` calls in adjoint passes.
   20: - **Zero abstraction tax.** Const generics, ZSTs, and GATs encode
   21:   capability and lifetime variance at the type level — there is no
   22:   runtime machinery to charge.
   23: 
   24: Where Atlas does not yet match legacy throughput, the gap is recorded in
   25: the per-crate migration chapters of this Part and governed by the parity
   26: test harness described in [Migration Validation](migration_validation.md).
